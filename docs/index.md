# Документация по репозиторию mts-hr

## Общее описание

Прототип для матчинга резюме кандидатов и вакансий, разработанный с использованием Streamlit и Yandex Cloud.

## Структура репозитория

```
├── resume_analyser/          # Основное приложение для анализа резюме
├── tools/                    # Вспомогательные инструменты
├── docs/                     # Документация (текущая директория)
├── .gitignore                # Игнорируемые файлы
├── constants.py              # Глобальные константы
├── pyproject.toml            # Зависимости проекта
├── README.md                 # Описание проекта
├── settings.py               # Настройки приложения
└── todo.md                   # Список задач и технического долга
```

## Основные компоненты

### resume_analyser
Основное приложение Streamlit для работы с резюме и вакансиями.

**Файлы:**
- `app.py` - Главный файл приложения с навигацией между экранами
- `constants.py` - Константы приложения
- `deps.py` - Зависимости приложения
- `navigation.py` - Навигация в интерфейсе
- `styles_old.py` и `styles.py` - Стили интерфейса
- `utils/` - Вспомогательные утилиты (логирование, трассировка)
- `warmup.py` - Инициализация состояния
- `screens/` - Экраны приложения:
  - `add_job.py` - Добавление вакансии
  - `add_resume.py` - Добавление резюме
  - `jobs_list.py` - Список вакансий
  - `resumes_list.py` - Список резюме
  - `links_input.py` - Ввод ссылок
  - `matching_jobs.py` - Подбор вакансий
  - `matching_resumes.py` - Подбор резюме
  - `result.py` - Результаты анализа

### tools
Вспомогательные инструменты для работы с Yandex Cloud.

**Файлы:**
- `run_gpt.py` - Класс ModelRunner для выполнения запросов к GPT моделям Yandex Cloud

### Конфигурационные файлы

- `constants.py` - Определяет базовую директорию проекта
- `settings.py` - Настройки для работы с Yandex Cloud (токен, folder_id)
- `pyproject.toml` - Зависимости проекта (Streamlit, Pydantic, Yandex Cloud SDK и др.)
- `README.md` - Основное описание проекта и технический долг
- `todo.md` - Список задач и технического долга

## Зависимости

Проект использует следующие основные зависимости:
- Streamlit (>=1.46.1,<2.0.0) - для веб-интерфейса
- pydantic-settings (>=2.10.1,<3.0.0) - для управления настройками
- yandex-cloud-ml-sdk (>=0.12.0,<0.13.0) - для работы с Yandex Cloud
- pandas (>=2.3.1,<3.0.0) - для работы с данными
- beautifulsoup4 (>=4.13.4,<5.0.0) - для парсинга HTML
- numpy, scipy - для численных вычислений

## Технический долг

Согласно README.md и todo.md, проект имеет следующие направления технического долга:

1. Создание lock-файла из pyproject.toml
2. Определение необходимости warmup
3. Объединение styles_old.py и styles.py
4. Хранение данных в PostgreSQL вместо файлов
5. Добавление логирования
6. Переписать с Streamlit на HTML + wf/cf
7. Вынести ModelRunner в отдельную библиотеку
8. Вынести JSON-файлы в git lfs
9. Вынести обработку текста вакансии
10. Вынести механизм ранжирования в отдельный модуль
11. Улучшить модульность, особенно для ранжирования
12. Упростить сложность ранжирования
13. Вынести секреты
14. Создать poetry файл

## Использование

Для запуска приложения необходимо:
1. Установить зависимости из pyproject.toml
2. Настроить переменные окружения YC_TOKEN и YC_FOLDER_ID
3. Запустить приложение командой `streamlit run resume_analyser/app.py`
